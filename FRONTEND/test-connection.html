Environment

Checking how this page is served…
❌ You are opening the file directly (file://)
Browsers block all fetch() calls from file:// to http:// — this is why you see "Cannot connect".
Fix: Install the Live Server extension in VSCode, right-click this file → Open with Live Server.
The URL in your browser should start with http://127.0.0.1:5500

Django Server

Checking Django on port 8000…
❌ Django is not running or not on port 8000
Open a terminal in your project folder and run: python manage.py runserver You should see: Starting development server at http://127.0.0.1:8000/
Then click Run Checks again.
Checking CORS headers…
❌ CORS is blocking your frontend
In your settings.py, make sure you have: INSTALLED_APPS = [ ... "corsheaders", ... ] MIDDLEWARE = [ "corsheaders.middleware.CorsMiddleware", # ← MUST be first "django.middleware.security.SecurityMiddleware", ... ] CORS_ALLOW_ALL_ORIGINS = True # ← add this line Then restart Django (Ctrl+C, then python manage.py runserver).

API Endpoints

Checking /api/login/ exists…
❌ /api/login/ returned 404 — the URL is not registered
Check your root urls.py (the one inside your project folder, e.g. core/urls.py or studentpal/urls.py): from django.urls import path, include urlpatterns = [ path("admin/", admin.site.urls), path("api/", include("accounts.urls")), # ← must be here ] Then check accounts/urls.py: urlpatterns = [ path("login/", LoginAPIView.as_view(), name="login"), path("register/", RegisterAPIView.as_view(), name="register"), ... ]
Checking /api/register/ exists…
